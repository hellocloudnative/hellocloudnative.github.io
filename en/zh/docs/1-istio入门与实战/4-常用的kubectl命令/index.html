常用的kubectl命令

2019-10-12
1 min read

由于istioctl的子命令create、get、replace、delete已经全部废弃，不再建

议使用，所以对资源的创建、更新、获取、删除，推荐使用kubectl来完成。

创建

创建Istio相关的新资源，使用参数–file(-f)指定配置文件，使用形式 如下:

kubectl create [flags]

示例如下:

$ kubectl create -f example-routing.yaml

更新

更新Istio相关的资源，当资源不存在时创建新资源，使用参数–file(-

f)指定配置文件，使用形式如下: kubectl apply [flags]

示例如下:

$ kubectl apply -f example-routing.yaml

获取

获取Istio相关的资源，使用形式如下: kubectl get [] [flags]

示例如下:

   $ kubectl get virtualservices

   $ kubectl get destinationrules

   $ kubectl get virtualservice bookinfo

删除

删除Istio相关的资源，可以使用参数–file(-f)指定配置文件，使用形式如下:

kubectl delete [ … ] [flags]

示例如下:

 $kubectl delete -f example-routing.yaml $ kubectl delete virtualservice bookinfo
